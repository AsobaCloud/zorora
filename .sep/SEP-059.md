# SEP-059: Deep Research Memory — feedback, chat persistence, scouting RAG

**Status:** Open
**Created:** 2026-03-22
**Project:** zorora

## Summary
Add three memory features to Deep Research: (1) thumbs up/down feedback on chat responses stored in SQLite, (2) persist chat threads to SQLite so they survive restarts, (3) inject completed scouting feasibility findings as internal sources during deep research aggregation.

## Motivation
Deep Research conversations and user feedback are lost on restart. Completed scouting feasibility analysis (production, trading, grid, regulatory, financial) is never reused as context for new research queries, missing an opportunity for knowledge compounding.

## Proposed Change
Add research_feedback and research_chat_history tables to zorora.db. Add feedback rating API endpoint. Persist chat threads on write. Add scouting_knowledge_sources() to aggregator that queries feasibility_results for keyword-relevant completed cases and returns them as Source objects with source_type='internal'.

## Acceptance Criteria
Feedback persists across restarts, chat threads reload on session resume, scouting feasibility findings appear as internal sources in deep research results when topically relevant.
