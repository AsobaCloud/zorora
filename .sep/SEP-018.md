# SEP-018: Comparative Query Support

**Status:** Open
**Created:** 2026-03-03
**Project:** zorora

## Summary
Add comparative query detection and subject-based variant generation so comparative queries produce balanced side-by-side analysis instead of single-subject monographs

## Motivation
Queries like 'comparison between lithium ion and other battery tech' produce monographs about only one subject because the pipeline decomposes by analytical angle not by comparison subjects

## Proposed Change
Add detect_comparison() to query_refiner, override variant generation for comparative queries, add comparison synthesis prompt template

## Acceptance Criteria
Comparative queries generate subject-specific search variants and synthesis output structured as comparison; non-comparative queries unchanged
