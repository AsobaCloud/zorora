# SEP-072: Enable finding clustering for thematic synthesis

**Status:** Open
**Created:** 2026-03-28
**Project:** zorora

## Summary
Activate the disabled finding clustering stage in run_deep_research so sources are organized into 8-15 thematic findings before synthesis, enabling multi-hop reasoning chains and structured evidence grouping

## Motivation
The synthesis pipeline has a finding clustering stage (synthesizer.py) that extracts thematic findings from sources before generating the final output. This is disabled at line 1047 of deep_research_service.py — the MVP goes straight to direct synthesis. Without clustering, the model receives a flat list of sources and must discover thematic structure on its own. Published research articles demonstrate multi-hop reasoning (Iran war → uranium security → French military → Sahel resources) that requires pre-organized evidence clusters.

## Proposed Change
Enable the finding clustering path in run_deep_research. Use the reasoning model to extract 8-15 thematic findings from ranked sources, each citing 1-5 directly-supporting sources. Pass findings (not raw sources) to the synthesis stage. Each finding gets confidence level (high/medium/low) based on source count. Test with Nehanda 7B to ensure clustering quality meets the thematic coherence bar.

## Acceptance Criteria
Finding clustering runs before synthesis. Sources are grouped into 8-15 thematic findings with confidence levels. Synthesis receives organized findings instead of flat source lists. Multi-hop reasoning chains are preserved across findings.
