# SEP-008: Add relevance filtering to deep research pipeline

**Status:** Open
**Created:** 2026-03-02
**Project:** zorora

## Summary
Add multi-layer relevance filtering (keyword scoring + newsroom fix + source budget enforcement) so only query-relevant sources reach clustering and synthesis. Zero new dependencies, with cross-encoder upgrade path.

## Motivation
Deep research returns completely irrelevant results. Newsroom has zero query filtering, no relevance scoring exists anywhere in the pipeline, max_sources budget is never enforced, and the 7B synthesis model hallucinates generic themes when fed irrelevant context.

## Proposed Change
Fix newsroom multi-word query handling, add keyword-based relevance scoring in new reranker module, enforce source budget from depth profiles, add relevance_score field to Source model

## Acceptance Criteria
Queries return only topically relevant sources; clustering produces 8-15 themed findings not 160 fallback; synthesis themes match query topic
