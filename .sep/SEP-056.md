# SEP-056: Paginate news-intel articles endpoint

**Status:** Open
**Created:** 2026-03-21
**Project:** zorora

## Summary
Frontend requests limit:10000, causing 1.1MB payload on every Digest tab load. Add server-side pagination with reasonable default limit and offset support.

## Motivation
Deployed Zorora at zorora.asoba.co transfers 1.1MB of JSON on every Digest tab switch because the articles endpoint returns all 3000 cached articles. This is unacceptable latency for users on the production deployment.

## Proposed Change
Add pagination params (limit, offset) to /api/news-intel/articles with a default limit of 50. Update frontend to use paginated fetches.

## Acceptance Criteria
Digest tab loads in under 500ms on second visit. Articles endpoint returns max 50 articles by default with total count for pagination.
