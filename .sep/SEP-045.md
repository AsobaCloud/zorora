# SEP-045: Feasibility Studies (5-Tab Analysis)

## Summary
Add a feasibility analysis panel within the Scouting section that provides 5 focused study types (Production, Trading, Grid Connection, Regulatory, Financial) for sites in the Feasibility kanban stage.

## Motivation
Investors need directional ROI assessment before committing to full diligence. Current workflow goes straight from scored site to full diligence report — there's no intermediate step for targeted "can this work?" analysis across specific dimensions. Feasibility studies fill this gap with lightweight, focused assessments that use available local data before triggering expensive research queries.

## Change Description

### Feasibility Panel
- Opens below kanban when a Feasibility-stage card is selected
- 5 sub-tabs, each producing an independent lightweight report:

#### 1. Production
- **Question:** Will this asset produce enough to justify the investment?
- **Inputs:** ACEC zone rankings (capacity factor, generation MWh/yr, LCOE), NASA POWER resource data, comparable assets from GEM tracker
- **Output:** Directional production estimate, comparable benchmarks, resource quality assessment

#### 2. Trading
- **Question:** Can you monetize through market participation?
- **Inputs:** SAPP DAM prices (hourly by node), Eskom TOU tariffs, Station Build Up (peaker dispatch)
- **Output:** Arbitrage spread analysis (24h price profile, seasonal), TOU differential, peaker displacement value
- **Charts:** 24h DAM price profile, monthly arbitrage revenue estimate (low/mid/high)

#### 3. Grid Connection
- **Question:** Is there physically room to connect?
- **Inputs:** GCCA MTS zones, supply areas, transmission GeoJSON
- **Output:** Nearest substation, voltage level, transmission zone, connection assessment

#### 4. Regulatory
- **Question:** Can you get licensed and permitted?
- **Inputs:** NERSA regulatory events, research sources, REIPPPP data
- **Output:** Licensing pathway, regulatory barriers, timeline estimate

#### 5. Financial
- **Question:** Does the directional ROI make sense?
- **Inputs:** Synthesis of all completed feasibility tabs + FRED FX rates, World Bank macro, commodity trends
- **Output:**
  - Directional IRR/payback range (not a financial model — "3-5 year payback at current spreads")
  - **Key risks** explicitly surfaced (currency, regulatory, demand, technology, competition)
  - **Unanswered data points** — what information is missing, what assumptions were made, sensitivity assessment
  - Confidence level based on how many other tabs are completed

### Report Format (per tab)
- Key finding (one paragraph)
- Data visualization (chart from local data where applicable)
- Directional conclusion: Favorable / Marginal / Unfavorable + confidence level
- Key risks and gaps specific to this dimension
- Sources

### Card Progress Tracking
- Each feasibility card shows completion: "3/5 done"
- Completed tabs are marked with conclusion (Favorable/Marginal/Unfavorable)
- Financial tab warns if run before other tabs are complete

### Backend
- New feasibility workflow orchestrating per-tab analysis
- Feasibility results stored per site per tab (allows re-running individual tabs)
- LLM synthesis for each tab using local data context + targeted research queries

## Dependencies
- SEP-042 (data parsers — needed for Trading, Grid, Financial tabs)
- SEP-044 (Scouting section — feasibility panel lives within Scouting)

## Success Criteria
- Feasibility panel opens for cards in Feasibility stage
- All 5 tabs produce focused reports with directional conclusions
- Financial tab surfaces key risks and unanswered data points
- Card progress indicator updates as tabs are completed
- Trading tab shows DAM price profile and revenue estimate charts
