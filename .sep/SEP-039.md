# SEP-039: Improve diligence search source quality and per-domain coverage

**Status:** Open
**Created:** 2026-03-12
**Project:** zorora

## Summary
Each diligence intent (6 domains) returns 11-14 raw sources, but after dedup and relevance filtering most domains end up with 0-2 usable sources. The LLM synthesis cites only 1-2 unique sources across all 6 sections because: (1) cross-intent dedup collapses sources that appear in multiple intents, (2) relevance scoring is keyword-overlap only (~36%) so generic educational articles score the same as specific regulatory docs, (3) many sources have empty titles ('No title') reducing citation quality.

## Motivation
The SEP-038 synthesis pipeline now correctly groups sources by domain and asks analytical questions per section, but the report quality is bottlenecked by source input: most sections say 'not covered in detail in the provided sources' because they literally have 0-1 sources. A Namibia solar diligence search returns good raw results per intent (IEA feed-in tariff pages, NERSA licensing docs, EIA assessments) but they get lost in dedup or score too low.

## Proposed Change
1. Preserve intent_domain through cross-intent dedup (keep both copies with different domains instead of dropping duplicates). 2. Add domain-aware relevance scoring that boosts sources matching their intent's domain terms. 3. Ensure each domain retains a minimum source budget (e.g., top 3 per domain) before global ranking cuts. 4. Fix empty-title sources from web search backends.

## Acceptance Criteria
1. Each of the 6 domain sections in a diligence report cites at least 2 distinct sources. 2. No domain section says 'not covered' when raw search returned relevant results for that domain. 3. Source titles are non-empty for all web search results.
