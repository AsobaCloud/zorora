# SEP-044: Scouting Section with Kanban Pipeline

## Summary
Add a new top-level "Scouting" section with tabbed kanban boards (Brownfield, Greenfield, BESS) for tracking sites through evaluation stages. Migrate brownfield pipeline and greenfield watchlist from Discovery (formerly Imaging) into Scouting.

## Motivation
Currently, pipeline tracking and site scoring are embedded in the Imaging view alongside map discovery. This bundles three concerns (discovery, tracking, analysis) into one view. Separating Scouting as its own section enables repeat tracking via kanban boards, cleaner Discovery view, and a natural place for feasibility studies (SEP-045).

## Change Description

### New Top-Level Section
- "Scouting" tab in main navigation (after Discovery)
- Three sub-tabs: Brownfield | Greenfield | BESS
- Each tab contains its own kanban board

### Kanban Board
- Five stages: Identified → Scored → Feasibility → Diligence → Decision
- Cards are draggable between stages
- Card content: site/asset name, location, key metric (capacity MW or BESS score), progress indicator
- Card action buttons vary by stage:
  - Identified: [Score] [Remove]
  - Scored: [Feasibility] [Diligence] [Focus on Map] [Remove]
  - Feasibility: shows completion (e.g., "3/5 done"), [Open Feasibility] [Diligence]
  - Diligence: [View Report] [Move to Decision]
  - Decision: [Go] [No-Go] label

### Data Migration
- Brownfield pipeline assets (currently in `pipeline_assets` table) become Brownfield kanban cards
- Greenfield watchlist sites (currently in `scouting_watchlist` table) become Greenfield kanban cards
- Add `scouting_stage` column to existing tables (or new unified `scouting_items` table)
- BESS kanban uses same schema with `item_type = 'bess'`

### Discovery View Changes
- Remove brownfield pipeline panel and greenfield watchlist panel from Discovery
- Keep greenfield click-to-score for quick evaluation
- Change [+ Pipeline] to [+ Scouting] on asset popups
- Change "Add to Watchlist" to "Add to Scouting" for scored greenfield sites

### API Changes
- `GET /api/scouting/items?type=brownfield|greenfield|bess&stage=X` — list items by type and stage
- `PUT /api/scouting/items/<id>/stage` — move item to different stage
- `POST /api/scouting/items` — add new item (from Discovery)
- `DELETE /api/scouting/items/<id>` — remove item
- Existing pipeline/watchlist endpoints become aliases or deprecated

## Dependencies
None — can be built independently of data parsers. Discovery rename (SEP-043) is coordinated but not blocking.

## Success Criteria
- Scouting tab visible in navigation with 3 sub-tabs
- Kanban boards render with 5 columns per tab
- Cards can be dragged between stages
- Existing pipeline assets and watchlist sites appear in appropriate kanban tabs
- Adding sites from Discovery populates Scouting kanban
- Discovery view no longer shows pipeline/watchlist panels
