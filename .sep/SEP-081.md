# SEP-081: Fix build-and-deploy: register new task definition with built image

**Status:** Open
**Created:** 2026-05-29
**Project:** zorora

## Summary
The deploy job never updates the running image, so production re-runs a stale pinned task-def image on every deploy.

## Motivation
build-and-deploy.yml's deploy job only runs 'aws ecs update-service --force-new-deployment' against the service's existing task definition (revision 9), which pins the immutable image tag prod-9196b81 from 2026-05-25. Newly built prod-<sha>/prod images are pushed to ECR but never referenced, so no code change since 2026-05-25 has actually deployed.

## Proposed Change
Replace the deploy job's single update-service step with: fetch the current task definition, set its container image to the freshly-built ECR image (needs.build-and-push.outputs.image_tag), register a new task-definition revision, update the service to that revision with force-new-deployment, and wait for services-stable.

## Acceptance Criteria
After pushing, the build-and-deploy run registers a task-def revision >9 whose image is the new prod-<sha>, the ECS service ona-zorora-prod runs that revision, and the rollout reaches steady state.
